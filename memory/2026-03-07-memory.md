## Changes Made

### Part 1: Connection Reliability Fixes

1. **stdout capture** (`src/bridge/__init__.py`)
   - Bridge `execute` handler now redirects `sys.stdout` to `StringIO` during `exec()`
   - `print()` output is captured and returned in the `"output"` field of the response
   - Previously, all stdout was silently lost — MCP handler received empty output

2. **Screenshot GET route** (`src/bridge/__init__.py`)
   - Added `/screenshot` to `do_GET` handler, delegating to `handle_viewport_screenshot`
   - `take_blender_screenshot` tool now works correctly (it calls GET `/screenshot`)

3. **Dynamic port fallback** (`src/bridge/__init__.py`)
   - Port binding now tries range 5000–5010 with `try/except OSError`
   - Lock file writes the actual port used, not hardcoded 5000
   - Multiple Blender instances can now coexist on different ports

4. **Universal ping health check** (`src/mcp_tools/blender_tools_handler_v2.py`)
   - All tool calls (except `get_blender_state`) now ping the bridge first
   - Provides immediate, descriptive error message on dead connections
   - Removed redundant per-tool ping from `execute_blender_code`

5. **Version bump** (`pyproject.toml`)
   - Updated from `1.0.0` to `2.5.0` to match server version
   - Added `numpy>=1.24.0` as explicit dependency
